Plugin overview

The NP_LMStopForumSpam plugin is a spam fighting plugin implementing the NP_LMCommentModerator SpamCheck API. It uses the Stop Forum Spam API to check if comments are spam.

Stop Forum Spam is a site that provides free access to a database of known forum and blog spammers. The database provides the spammers email addresses, IP addresses, usernames, spamming frequency, and, in some cases, evidence of their spam.

The NP_LMStopForumSpam plugin just implements the SpamCheck API events. For this plugin to do any work is it required that the NP_LMCommentModerator plugin is installed and used for comment moderation. The NP_LMCommentModerator plugin can be downloaded from the NP_LMCommentModerator plugin page.

Installation

  • Upload the NP_LMStopForumSpam.php file and the lmstopforumspam directory from the zip file to the Nucleus CMS plugins directory (nucleus/plugins) on your web server.
  • Go to the Plugins page in the admin area. You should be able to select the plugin in the dropdown list under the "Install New Plugin" section and press the "Install Plugin" button.

Upgrade

  • Take a backup of the current plugin files. You will need these if you want to rollback to the previous version after the upgrade.
  • Take a backup of the database used for you Nucleus installation.
  • Replace the old plugin files with the new ones. Do not uninstall the plugin.
  • If a plugin data upgrade is needed, will an upgrade plugin data option be available on the NP_LMStopForumSpam plugin admin page. Choose this option to upgrade the plugin data.
  • After the plugin data upgrade has been performed may a rollback option and a commit option be available on the NP_LMStopForumSpam plugin admin page. Not all upgrades support the rollback and commit option.
  • If you are not pleased with the new version of the plugin you may use the rollback option to rollback the plugin data upgrade. After the rollback of the plugin data upgrade you have to replace the new plugin files with the old plugin files.
  • If you are pleased with the new version of the plugin you should use the commit option to commit the plugin data upgrade. After the commit of the plugin data upgrade will any rollback and commit options disappear.

Configuration

The NP_LMStopForumSpam plugin has the following options:

  • Check IP against StopForumSpam - Should the comment author IP be checked against the StopForumSpam database.
  • Check EMail against StopForumSpam - Should the comment author email address be checked against the StopForumSpam database.
  • Check Name against StopForumSpam - Should the comment author name be checked against the StopForumSpam database. This option is set to No by default. It is not recommended to enable this check.
  • Enable spam check against StopForumSpam - Should the plugin spam check comments against the StopForumSpam database.

Usage

After the plugin is installed will it automatically be used by the NP_LMCommentModerator plugin to check for spam comments.

Stop Forum Spam

This plugin does not implement Stop Forum Spam submissions based on moderator comment categorization.

The reason for this is that the NP_LMCommentModerator plugin does not verify EMail addresses of non-member authors, and Stop Forum Spam requires that any EMail addresses submitted has been verified.

Compability

This version of the LMStopForumSpam plugin is tested against version 3.66 of LMNucleus CMS. The minimum version of Nucleus CMS needed for using this plugin is 3.60. The minimum version of the NP_LMCommentModerator plugin needed by this plugin is 1.0.0.
